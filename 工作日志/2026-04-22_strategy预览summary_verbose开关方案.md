# 2026-04-22 strategy 预览 summary_verbose 开关方案

## 1. 背景

截至 `2026-04-22`，`strategy` 单日期预览入口已经具备：

- [tools/preview_strategy.py](C:/Users/ai/Desktop/可转债多因子/tools/preview_strategy.py)
- 观察名单输入与聚焦视角
- 环境分数、因子权重、目标组合、对齐摘要、提示信息的终端输出

当前问题不在于“信息不够”，而在于默认输出已经开始偏重：

- 日常实跑更关心“今天能不能看、看什么、看哪几只”
- 但当前默认输出会把完整环境分段、因子分段、目标组合表、对齐摘要、提示一起铺开
- 当用户只是做每日联调或快速查看时，阅读成本开始变高

因此，本轮补一个真正可用的 `summary only / verbose` 双视图，而不是只加一个无实质差异的参数开关。

## 2. 目标

本轮目标：

1. 让 `strategy` 预览默认更适合日常实跑。
2. 保留完整诊断能力，避免因为压缩输出而失去联调抓手。
3. 不改变现有 `strategy` 计算链路，只调整终端展示层。

用户体验目标：

1. 默认运行时看到摘要模式输出。
2. 如果需要完整细节，可显式加 `--verbose`。
3. 观察名单视角继续可用，并随摘要/详细模式一起调整展示粒度。

## 3. 设计原则

### 3.1 只改展示层，不改策略计算口径

本轮不改变：

- `StrategyService`
- `StrategyEngine`
- 因子标准化口径
- 目标组合构建逻辑

本轮只调整 [tools/preview_strategy.py](C:/Users/ai/Desktop/可转债多因子/tools/preview_strategy.py) 的入口参数和渲染层组织方式。

### 3.2 默认值服务“日常实跑”

既然清单里明确提出“避免当前信息量对日常实跑偏重”，本轮建议默认行为改为：

- 默认：摘要模式
- `--verbose`：详细模式

同时保留显式 `--summary-only`，方便脚本或调用方明确表达意图。

### 3.3 详细模式保留当前联调抓手

详细模式不应被削弱，仍然保留：

- 环境分数分段输出
- 因子权重分段输出
- 完整目标组合表
- 完整观察名单表
- 对齐摘要
- 提示 / 数据质量信息

也就是说，详细模式应尽量接近当前默认输出。

## 4. 入口设计

建议在 CLI 中新增一组互斥参数：

- `--summary-only`
- `--verbose`

规则如下：

1. 如果显式传 `--verbose`，输出详细模式。
2. 如果显式传 `--summary-only`，输出摘要模式。
3. 如果两者都不传，默认走摘要模式。

交互式模式不再额外询问视图类型，避免增加用户操作负担，只在启动说明中明确：

- 默认摘要模式
- 如需完整诊断，请改用 `--verbose`

## 5. 摘要模式设计

摘要模式不是简单删行，而是重组为更紧凑的终端读法。

建议结构如下：

### 5.1 基本信息

保留：

- `trade_date`
- `fetch_policy`
- `refresh_requested`
- `data_quality_status`
- `history window`
- `eligible / selected / cash`

但计数信息收口为更紧凑的一行。

### 5.2 环境摘要

不再逐行展开为一个单独大段，而是压缩成一行：

- `equity`
- `bond`
- `trend`

### 5.3 因子权重摘要

同样压缩成一行，不再逐行拆开。

### 5.4 目标组合简表

摘要模式下只保留：

- `rank`
- `cb_code`
- `target_weight`

不再默认展示 `total_score`，减少宽表阅读负担。

### 5.5 观察名单简表

如果指定了观察名单，则摘要模式下展示一个更紧凑的版本，只保留：

- `代码`
- `观察状态`
- `组合排序`
- `目标权重`
- `总分`
- `剔除原因`

### 5.6 提示策略

摘要模式下：

- 不默认展示 `alignment_summary`
- 不默认展示常规 `data_quality_hints`
- 保留真正有必要的 `notes`
- 如果 `data_quality_status != 正常`，再把相关 warning / hints 一起展示出来

这样既减少噪音，又不至于把风险提示吞掉。

## 6. 详细模式设计

详细模式保留当前主结构：

- 基本信息
- 观察名单详细表
- 环境分数
- 因子权重
- 目标组合
- 对齐摘要
- 提示

也就是说，本轮实现的关键不是“发明一套新 verbose”，而是把当前默认输出收口为 `verbose`。

## 7. 测试要求

本轮至少补下面几类测试：

1. 默认 `main(...)` 输出摘要模式。
2. `--verbose` 会切回完整详细模式。
3. 摘要模式不再输出 `alignment_summary`。
4. 详细模式继续输出环境分数分段、因子权重分段和对齐摘要。
5. 观察名单视角在摘要/详细模式下都能工作，但列宽不同。

## 9. 结论

本轮最稳的落地方式是：

1. 让 `preview_strategy` 默认输出摘要模式
2. 用 `--verbose` 保留当前完整联调输出
3. 把观察名单表和组合表都拆成摘要版 / 详细版两种渲染
4. 继续保持策略计算链路完全不变

这样可以直接改善每天实际使用时的阅读体验，同时不牺牲问题定位能力。
