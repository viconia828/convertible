## 1. 架构哲学

### 1.1 核心分离原则

整个系统拆成三个**可独立运行**的层：

```
┌─────────────────────────────────────────────┐
│            调参系统（Tuner）                  │
│   把回测当黑盒：给参数 → 拿结果 → 评好坏     │
│   固定参数回测 / 在线学习调权 / 对比报告      │
│                                              │
│   ┌───────────────────────────────────────┐  │
│   │          回测引擎（Backtester）        │  │
│   │   时间循环：推日期、算净值、扣成本      │  │
│   │   每个再平衡日调用主策略拿目标组合      │  │
│   │                                       │  │
│   │   ┌───────────────────────────────┐   │  │
│   │   │      主策略（Strategy）        │   │  │
│   │   │   纯函数：数据 + 参数 → 组合   │   │  │
│   │   │   不知道自己在实盘还是回测      │   │  │
│   │   └───────────────────────────────┘   │  │
│   └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

**为什么这样拆：**

| 问题 | 旧架构的痛点 | 新架构的解法 |
|------|-------------|-------------|
| 策略和回测纠缠 | 策略函数里塞了净值计算、滑点、持有期 | 策略是纯函数，不碰执行细节 |
| 调参和策略纠缠 | 在线学习嵌在策略内部，无法换优化器 | 调参是外层编排，策略对它无感知 |
| 实盘复用困难 | 回测逻辑和交易逻辑混在一起 | 实盘只需调用主策略同一个函数 |
| 调试困难 | 一出错不知道是策略错还是回测错 | 三层独立，逐层定位 |

### 1.2 三层职责边界

| 层 | 输入 | 输出 | 知道什么 | 不知道什么 |
|----|------|------|----------|-----------|
| **主策略** | 当日截面数据 + StrategyConfig | 目标组合权重 | 因子怎么算、环境怎么读、风险怎么约束 | 今天是回测还是实盘、上一期持仓是什么、净值是多少 |
| **回测引擎** | 历史数据 + StrategyConfig + 起止日期 | 净值曲线 + 交易记录 + 持仓历史 | 时间怎么推进、成本怎么扣、持有期怎么管 | 策略内部逻辑、因子是什么 |
| **调参系统** | 固定/在线学习参数 + 回测引擎引用 | 对比报告 + 建议 | 怎么跑基线、怎么在线调权、怎么对比效果 | 策略内部逻辑、回测内部实现 |

### 1.3 共享层

三层共享两样东西，**且仅共享这两样**：

1. **StrategyConfig** — 统一参数对象，所有可调参数的单一来源
2. **DataManager** — 统一数据接口，双通道调度

```
StrategyConfig ─────┬──→ 主策略
                    ├──→ 回测引擎
                    └──→ 调参系统

DataManager ────────┬──→ 主策略（通过回测引擎传入当日数据切片）
                    ├──→ 回测引擎（拉取全量历史）
                    └──→ 调参系统（不直接用，通过回测引擎间接使用）
```

### 1.4 策略信条

| 序号 | 信条 | 含义 |
|------|------|------|
| 1 | 以慢打快 | 低换手、低频决策、低频调仓 |
| 2 | 环境调权重，不切策略 | 不做择时，只做风格偏移 |
| 3 | 因子决定想买什么，风险决定能买多少 | 风险约束独立于因子打分 |
| 4 | 顶部是过程，不是点 | 不做精确逃顶，做渐进减仓 |
| 5 | 先活下来，再谈收益 | risk > execution > factor |
| 6 | 策略是纯函数 | 同样的输入永远给同样的输出 |
| 7 | 债性保护是天然的安全垫 | 可转债有债底，不需要在熊市清仓，只需要让组合偏向债性 |

### 1.5 不做什么

- 不做日内交易
- 不做择时开关（全仓/空仓切换）
- 不做反向操作（风险只降仓，不反手做空）
- 不在策略函数内部记录状态
- 不追求拟合历史最优参数
- 不做熊市清仓（可转债的债性保护天然提供安全边际，通过因子权重偏移应对，不通过仓位择时）

## 2. 主策略设计

### 2.1 核心定义

主策略是一个**无状态纯函数**：

```
target_weights = strategy(snapshot, config)
```

- **snapshot**：当日市场截面数据（可转债行情、正股行情、指数、宏观指标）+ 足够的历史窗口
- **config**：StrategyConfig（所有参数）
- **target_weights**：`Dict[cb_code, float]`，目标组合权重，和为 1.0（或 < 1.0 表示部分现金）

**纯函数意味着**：
- 没有 `self.xxx` 状态
- 不读取上一期持仓（不知道自己持有什么）
- 不读取净值（不知道自己赚了还是亏了）
- 不读取当前日期在全回测中的位置（不知道是第几轮）
- 同一份 snapshot + 同一份 config → 永远输出同一个 target_weights

**策略不做的事**（由回测引擎负责）：
- 不管冷却期（"这只刚买入不能卖"）
- 不管换手率（"这次调仓太多要缩减"）
- 不管滑点和手续费
- 不管实际持仓和目标之间的差异如何弥合

### 2.2 Snapshot 结构

策略函数只接收一个 `snapshot` 对象，里面包含做一次决策所需的全部信息：

```
snapshot = {
    "trade_date": date,                     # 当日日期
    "universe": List[str],                  # 可交易转债列表（已过滤）
    "cb_daily": DataFrame,                  # 可转债日频数据，包含足够历史窗口
    "stock_daily": DataFrame,               # 正股日频数据
    "index_daily": DataFrame,               # 指数日频数据
    "macro_daily": DataFrame,               # 宏观指标日频数据
    "cb_info": DataFrame,                   # 转债静态信息
}
```

**关键约定**：
- `cb_daily` 等 DataFrame 包含的历史深度由 `config.data.lookback_days` 决定（默认504个交易日，约2年）
- 数据截止到 `trade_date`（含），不包含未来数据
- 数据已经过清洗和对齐（由 DataManager 保证）
- `universe` 已经过入池过滤（由调用方负责）

### 2.3 策略内部流水线

```
strategy(snapshot, config):
    ├── Step 1: 环境计算
    │   env = compute_environment(snapshot, config)
    │   → {"equity_strength", "bond_strength", "trend_strength"}
    │
    ├── Step 2: 因子计算
    │   factors = compute_factors(snapshot, config)
    │   → DataFrame: [cb_code × 5因子分数]
    │
    ├── Step 3: 因子权重
    │   weights = compute_factor_weights(env, config)
    │   → {"value": 0.23, "carry": 0.21, ...}
    │
    ├── Step 4: 综合打分
    │   scores = factors @ weights
    │   → Series: [cb_code → 总分]
    │
    ├── Step 5: 组合构建
    │   raw_portfolio = build_portfolio(scores, config)
    │   → Dict[cb_code → weight]
    │
    ├── Step 6: 风险约束
    │   final_portfolio = apply_risk_constraints(raw_portfolio, snapshot, config)
    │   → Dict[cb_code → weight]
    │
    └── return final_portfolio
```

每一步都是纯函数，只依赖 snapshot 和 config，不依赖任何外部状态。

### 2.4 风险约束在策略内部还是外部？

**在策略内部。** 原因：

- 风险约束（单债上限、行业分散、波动率控制）是策略逻辑的一部分——它定义了"我想要什么样的组合"
- 策略输出的 target_weights 应该是一个**已经考虑了风险偏好**的目标
- 回测引擎不应该替策略做风险决策

**但回撤控制有个特殊问题**：策略是无状态的，它不知道当前回撤是多少。

**解法**：通过 snapshot 传入回撤信息，但包装为"市场指标"而非"组合状态"：

```
snapshot 额外字段（可选，由回测引擎注入）：
    "portfolio_hint": {
        "recent_drawdown": float,       # 近期回撤幅度（如果有的话）
        "recent_volatility": float,     # 近期波动率
        "equity_beta": float            # 近期股性Beta
    }
```

- 回测模式：引擎从组合历史中计算这些值，注入 snapshot
- 实盘模式：从实际账户数据计算这些值，注入 snapshot
- 首次运行（无历史）：不注入，策略使用默认值（不降仓）

**这样策略仍然是纯函数**——它只是多了一些可选输入，不改变无状态的本质。

## 3. 环境层设计

### 3.1 定位

环境层是主策略内部的第一步，回答：**当前市场处于什么状态？**

它只为因子权重调整提供输入，不做买卖决策。

### 3.2 三维环境向量

```
env = {
    "equity_strength": float,   # 股性环境强度 [0, 1]
    "bond_strength": float,     # 债性环境强度 [0, 1]
    "trend_strength": float     # 趋势环境强度 [0, 1]
}
```

三个维度不互斥，可以同时为高。

### 3.3 各维度具体计算

#### equity_strength（股性环境）

衡量权益市场是否活跃、是否有利于转债的进攻性。

| 子指标 | 计算方式 | 含义 |
|--------|----------|------|
| csi300_mom_60 | 沪深300过去60日收益率 | 中期股市动量 |
| csi300_above_ma120 | 沪深300是否在120日均线上方（0或1） | 中期趋势方向 |
| amount_ratio | 沪深300近20日成交额 / 近60日成交额 | 资金活跃度 |

```
raw_equity = 0.4 * percentile(csi300_mom_60, 历史滚动分布)
           + 0.3 * csi300_above_ma120
           + 0.3 * clip(amount_ratio, 0.5, 1.5) 归一化到 [0,1]
equity_strength = EMA(raw_equity, span=20)
```

#### bond_strength（债性环境）

衡量债市环境是否有利于转债的防守性。

| 子指标 | 计算方式 | 含义 |
|--------|----------|------|
| treasury_chg_60 | 10年国债收益率60日变化（取负，下行为正） | 利率趋势 |
| credit_spread_pct | 信用利差在历史中的滚动分位数 | 信用环境宽松度 |
| bond_index_mom_20 | 中证全债指数20日收益率 | 短期债市动量 |

```
raw_bond = 0.4 * percentile(-treasury_chg_60, 历史滚动分布)
         + 0.3 * (1 - credit_spread_pct)
         + 0.3 * percentile(bond_index_mom_20, 历史滚动分布)
bond_strength = EMA(raw_bond, span=20)
```

#### trend_strength（趋势环境）

衡量可转债市场本身的趋势存在性和可持有性。最核心的环境变量。

| 子指标 | 计算方式 | 含义 | 权重 |
|--------|----------|------|------|
| 趋势存在性 | 转债等权指数 ADX(14) / 40，clip到[0,1] | 市场是否有方向 | 0.4 |
| 趋势持续性 | 过去60日中 close > MA60 的天数占比 | 趋势持续了多久 | 0.4 |
| 趋势稳定性 | 1 - 转债等权指数20日波动率的历史滚动分位数 | 趋势是否平稳 | 0.2 |

```
raw_trend = 0.4 * trend_exist + 0.4 * trend_persist + 0.2 * trend_stable
trend_strength = EMA(raw_trend, span=10)
```

### 3.4 所有分位数计算规则

- 使用滚动历史窗口（至少252日），不使用全样本
- ADX 需要至少14日数据，不足时返回 0.5（中性值）
- 转债等权指数若数据源不提供，需自行计算（全市场转债等权日收益率的累积净值）

### 3.5 环境层的使用边界

| 允许 | 禁止 |
|------|------|
| 控制因子权重偏移 | 直接决定买卖 |
| 控制持仓周期弹性 | 做全仓/空仓开关 |
| 控制卖出宽松度 | 判断顶部/底部 |
| 作为调参系统的检索维度 | 直接改变风险参数 |

## 4. 因子体系设计

### 4.1 五因子结构

| 因子名 | 符号 | 信息维度 | 方向 |
|--------|------|----------|------|
| Value | double_low | 是否便宜 | 越低越好 → 取负后越大越好 |
| Carry | ytm | 是否有债性收益 | 越高越好 |
| Structure | premium_shape | 股债结构是否合理 | 钟形函数，中间最优 |
| Trend | momentum_60 | 是否在上涨 | 越高越好 |
| Stability | volatility_60 | 是否好持有 | 越低越好 → 取负后越大越好 |

**核心原则**：所有因子最终输出方向统一为**越大越好**。

### 4.2 各因子详细计算

#### Value（价值因子）

```
double_low = cb_close + premium_rate
value_raw = -double_low
value_score = zscore(value_raw, 当日全市场截面)
```

- 双低值越低 → 越便宜 → 取负后越大
- zscore 在当日全市场截面上计算（不是时间序列）
- 极值处理：计算 zscore 前先做 winsorize（1%/99% 分位截断）

#### Carry（债性因子）

```
carry_score = zscore(ytm, 当日全市场截面)
```

- YTM 越高 → 纯债角度越划算
- YTM 为负的券（深度价内），carry_score 自然很低
- 极值处理：同上 winsorize

#### Structure（结构因子）

```
premium_center = 20        # 溢价率最优中心（可配置）
premium_width = 15         # 钟形半宽（可配置）

structure_raw = exp(-0.5 * ((premium_rate - premium_center) / premium_width) ^ 2)
structure_score = percentile(structure_raw, 当日全市场截面) / 100
```

- 高斯钟形函数：溢价率在 `center ± width` 范围内得分最高
- 溢价率过低（深度价内，可能面临强赎）和过高（纯债性，缺乏弹性）都得分低

#### Trend（趋势因子）

```
momentum_60 = cb_close / cb_close.shift(60) - 1
trend_score = percentile(momentum_60, 当日全市场截面) / 100
```

- 过去60个交易日的价格动量
- percentile 归一化，避免极端值主导

#### Stability（稳定因子）

```
returns = cb_close.pct_change()
vol_60 = rolling_std(returns, window=60) * sqrt(252)
stability_score = -percentile(vol_60, 当日全市场截面) / 100 + 1
```

- 波动率越低 → 越稳定 → 取负 + 偏移使方向变为"越大越好"

### 4.3 因子前置过滤（入池条件）

在计算因子之前，先做标的过滤，生成**可交易池（universe）**：

| 过滤条件 | 阈值 |
|----------|------|
| 上市天数 | >= 30 交易日 |
| 剩余规模 | >= 0.5 亿 |
| 近20日均成交额 | >= 200万 |
| 已公告强赎 | 排除 |
| 已触发回售 | 排除 |
| 当日停牌 | 排除 |

### 4.4 因子输出格式

每次调用，因子层输出一个 DataFrame：

| 列 | 类型 |
|----|------|
| cb_code | str |
| value_score | float |
| carry_score | float |
| structure_score | float |
| trend_score | float |
| stability_score | float |

所有分数已归一化，方向统一为越大越好。

## 5. 环境 → 因子权重映射

### 5.1 核心原则

环境的输出不直接决定买卖，而是调整五个因子的相对权重。

### 5.2 基准权重

| 因子 | 基准权重 | 理由 |
|------|----------|------|
| Value | 0.25 | 核心因子，可转债alpha主要来源 |
| Carry | 0.20 | 提供安全边际 |
| Structure | 0.20 | 保证股债结构合理 |
| Trend | 0.20 | 捕捉中期趋势 |
| Stability | 0.15 | 辅助因子，降低组合波动 |

### 5.3 偏移矩阵

```
eq_dev = equity_strength - 0.5
bd_dev = bond_strength - 0.5
tr_dev = trend_strength - 0.5

#                   Value  Carry  Struct  Trend  Stab
shift_matrix = [
    [-0.05, -0.03,  0.00, +0.06, -0.03],   # equity_dev
    [+0.04, +0.06, +0.02, -0.03, -0.02],   # bond_dev
    [-0.05, -0.02,  0.00, +0.07, -0.03],   # trend_dev
]

adjustment = eq_dev * shift_matrix[0]
           + bd_dev * shift_matrix[1]
           + tr_dev * shift_matrix[2]

target_weights = normalize(clip(base_weights + adjustment, min=0.05, max=0.40))
```

**注意**：主策略是无状态的，所以**没有跨期平滑**（`0.8 * prev + 0.2 * target` 这种逻辑不在策略内部）。权重平滑是回测引擎的事——它记住上一期的 target_weights，在执行时做平滑。

## 6. 组合构建

### 6.1 打分

```
total_score[i] = Σ (factor_score[i][k] * weight[k])
```

### 6.2 持仓数量

- 默认持仓数：20（可配置，范围10-30）
- 选取 total_score 排名前N

### 6.3 权重分配

**主方案：Softmax**

```
raw_weight[i] = exp(total_score[i] / temperature)
target_weight[i] = raw_weight[i] / sum(raw_weight)
```

**备选方案：等权**（MVP 阶段使用）

### 6.4 组合约束（静态，在策略内部执行）

| 约束 | 阈值 |
|------|------|
| 单债最大权重 | 10% |
| 前5大持仓权重 | <= 40% |
| 单行业最大权重 | 25% |
| 最低持仓数 | 10只 |

超限后按比例缩放 + 重新归一化。

## 7. 风险约束（策略内部）

### 7.1 策略内部能做的风险约束

策略内部能做**基于当日截面数据**的风险约束：

| 约束类型 | 实现方式 | 为什么在策略内部 |
|----------|----------|----------------|
| 静态集中度约束 | 单债/行业/top5上限 | 纯截面逻辑，不需要历史状态 |
| 波动率偏好 | 基于个券波动率压低高波动券权重 | 来自因子分（Stability） |
| 股性暴露偏好 | 基于溢价率结构限制高Delta券占比 | 来自因子分（Structure） |
| 组合波动率约束 | 基于 snapshot.portfolio_hint 中的 recent_volatility | 可选输入 |
| 回撤响应 | 基于 snapshot.portfolio_hint 中的 recent_drawdown | 可选输入 |

### 7.2 回撤响应逻辑（基于 portfolio_hint）

```
如果 snapshot 中有 portfolio_hint：
    drawdown = portfolio_hint["recent_drawdown"]
    trend = env["trend_strength"]

    回撤区间        缩放系数
    < 5%           scale = 1.0
    5% ~ 10%       scale = 0.9
    10% ~ 15%      scale = 0.7
    > 15%          scale = 0.5

    趋势修正：scale += 0.15 * trend_strength，不超过 1.0

    final_weights *= scale
    （权重和 < 1.0 的部分 = 现金）

如果没有 portfolio_hint：
    不做回撤缩放（scale = 1.0）
```

### 7.3 顶部风险评估（Top Risk）

基于当日截面数据可计算的市场过热信号：

| 子信号 | 计算方式 |
|--------|----------|
| 过热度 | 转债指数偏离MA120的程度 |
| 动量衰减 | 近20日动量 vs 近60日动量 |
| 失稳度 | 近10日大阳大阴(涨跌>3%)次数 / 10 |
| 结构破坏 | 高价券(>130)占比变化（与20日前对比） |

```
top_risk = 0.3 * 过热度 + 0.3 * 动量衰减 + 0.2 * 失稳度 + 0.2 * 结构破坏
top_risk ∈ [0, 1]

如果 top_risk > 0.6：额外缩减仓位 10%~20%
```

### 7.4 策略不做的风险控制（由回测引擎或实盘系统负责）

| 风险控制 | 为什么不在策略内 | 谁负责 |
|----------|----------------|--------|
| 极端风险熔断（单日暴跌） | 需要逐日监控，不是再平衡日逻辑 | 回测引擎（快系统） |
| 冷却期 | 需要知道"什么时候买的"，是状态 | 回测引擎 |
| 换手率控制 | 需要知道上一期持仓，是状态 | 回测引擎 |
| 最大回撤熔断 | 需要实时净值监控 | 回测引擎（快系统） |

## 8. 回测引擎设计

### 8.1 核心定义

回测引擎是一个**时间循环器**，它：

1. 管理日历和时间推进
2. 在每个再平衡日调用主策略拿目标组合
3. 处理目标组合和实际持仓之间的差异（换手、冷却、成本）
4. 每日更新净值和风险指标
5. 向策略注入 portfolio_hint

**回测引擎不知道**策略内部是怎么算的，它只看到：`strategy(snapshot, config) → target_weights`。

### 8.2 回测引擎的状态

与主策略不同，回测引擎**有状态**，它维护：

```
BacktestState:
    # 时间
    current_date: date
    rebalance_dates: List[date]

    # 持仓与净值
    holdings: Dict[str, float]              # cb_code → 持有数量
    actual_weights: Dict[str, float]        # cb_code → 实际权重
    cash: float
    nav: float
    nav_history: List[float]
    peak_nav: float

    # 风险指标（用于注入 portfolio_hint）
    drawdown: float
    realized_vol: float
    equity_beta: float

    # 执行管理
    cooldown_map: Dict[str, date]           # cb_code → 冷却期截止日
    prev_target_weights: Dict[str, float]   # 上一期目标权重（用于平滑）

    # 记录
    trade_history: List[Dict]
    turnover_history: List[float]
```

### 8.3 回测主循环

```
for each trading_day:

    ──── 每日都做 ────

    (1) 按当日收盘价更新持仓市值
    (2) 计算当日组合收益，更新 nav
    (3) 更新 peak_nav、drawdown、realized_vol、equity_beta

    (4) 快系统检查（极端风险保护）：
        if 单日组合跌幅 > 3%：
            次日总仓位不超过当前的70%
        if 连续3日累计跌幅 > 5%：
            总仓位降至50%以下
        if 转债指数单日跌幅 > 5%：
            暂停新建仓，只允许减仓

    ──── 仅再平衡日做 ────

    (5) 准备 snapshot：
        - 从 DataManager 获取截止当日的数据切片
        - 生成 universe（可交易池过滤）
        - 注入 portfolio_hint = {drawdown, realized_vol, equity_beta}

    (6) 调用主策略：
        target_weights = strategy(snapshot, config)

    (7) 权重平滑（回测引擎负责）：
        if prev_target_weights 存在：
            smoothed = 0.8 * prev_target_weights + 0.2 * target_weights
        else：
            smoothed = target_weights
        保存 prev_target_weights = target_weights

    (8) 换手率控制：
        delta = smoothed - actual_weights
        turnover = sum(abs(delta)) / 2
        if turnover > max_turnover：
            delta *= max_turnover / turnover

    (9) 冷却期过滤：
        对于 delta < 0（卖出）的标的：
            if 在冷却期内 且 不是风险强制减仓：跳过
        对于买入：不受冷却期限制

    (10) 再平衡阈值过滤：
         abs(delta) < 1% 的标的不操作

    (11) 计算交易成本：
         买入/卖出佣金 0.02%
         滑点 0.05%（日均成交额 < 1000万的小券上浮至 0.10%）

    (12) 执行调仓，更新 holdings、actual_weights、cash
         新买入标的设置冷却截止日（+10个交易日）

    ──── 每日收尾 ────

    (13) 记录当日快照
```

### 8.4 再平衡频率

| 频率 | 适用 |
|------|------|
| 周频（每周五） | 趋势明确时 |
| 双周频 | 默认推荐 |
| 月频（每月最后交易日） | 保守版 |

### 8.5 回测引擎的输出

```
BacktestResult:
    nav_series: Series              # 日频净值
    trade_log: DataFrame            # 每笔交易明细
    holdings_history: DataFrame     # 每日持仓明细
    turnover_history: Series        # 每次调仓的换手率
    risk_history: DataFrame         # 日频风险指标
    portfolio_hint_history: DataFrame  # 每次注入的 hint
    config_used: StrategyConfig     # 使用的参数快照
```

### 8.6 回测引擎作为黑盒的接口

对调参系统而言，回测引擎只暴露一个接口：

```
def run(config: StrategyConfig, start_date, end_date) -> BacktestResult
```

调参系统不需要知道回测内部怎么循环，只需要：给参数 → 拿结果。

## 9. 调参系统设计

### 9.1 核心定义

调参系统是**最外层编排器**，但它刻意保持简单。核心思路：**固定参数跑通，在线学习可选增强。**

调参系统只做三件事：

1. 用固定参数跑一遍回测（Baseline）
2. 用在线学习调权跑一遍（Online Learning）
3. 对比两条净值曲线，人为判断是否开启在线学习

**调参系统不知道**策略内部怎么计算，也不知道回测引擎内部怎么循环。它只看到：`run(config) → result`。

### 9.2 为什么这样设计（而不是更复杂）

原设计包含四种调参模式（固定参数、网格搜索、Walk-forward、在线学习）和一套防过拟合检验框架。经过评估，决定砍到最简。理由如下：

#### 去掉网格搜索的原因

这个策略的五因子、权重、持仓数量都是人为设计的，背后有明确的投资逻辑。如果需要机器帮你搜最优参数，说明对策略逻辑本身没有信心——这时候该做的是重新思考策略，而不是加一个优化器。

网格搜索最大的问题不是过拟合（虽然这也是问题），而是**它会给你虚假的安全感**：一组参数在回测中夏普 1.8，你以为找到了圣杯，实际上只是在有限样本上碰巧最优。

#### 去掉 Walk-forward 独立模式的原因

Walk-forward 的本质是"用过去训练，用未来验证"。在线学习天然就是这个结构——每个再平衡点只看过去的因子表现来调权重。把 Walk-forward 做成独立模块是多余的工程复杂度。

#### 去掉复杂评价函数和防过拟合框架的原因

评价函数简化为直看夏普和最大回撤，不需要综合加权评分。防过拟合的最好防线不是复杂的检验框架，而是**"调整幅度本来就小，错了也亏不了多少"**。

#### 在线学习为什么不容易过拟合

在线学习调的是因子权重，不是因子本身。它做的事情非常有限：

```
base_weights = [0.25, 0.20, 0.20, 0.20, 0.15]   # 固定的
adjustment  = [+0.02, -0.01, 0.00, +0.03, -0.01] # 在线学习给的偏移
               ↑ 被 clip 到 ±3%，被平滑
```

它不改因子公式，不改持仓数量，不改风险参数。调整幅度被三层锁死（learning_rate=0.10、max_adjustment=0.03、权重平滑），最终一个因子从 0.20 变到 0.22，这种量级的偏移很难构成过拟合。

对比传统过拟合的场景——在 1000 个技术指标里选最好的 5 个、在 100 组参数里选最优——在线学习做的事量级小得多。

#### 真正的风险及应对

| 风险 | 说明 | 应对 |
|------|------|------|
| 样本不够 | 可转债有效历史约 8 年（2018-2026），top-50 检索去重后可能只剩 25-30 个独立样本，IC 估计标准误大 | 提高 min_similar_samples 到 30，不够就回退固定权重 |
| 环境误判 | equity_strength 在 0.48 和 0.52 之间跳动，检索到不同历史样本 | 环境层的 EMA 平滑缓解；最终调整幅度被 clip 限死 |
| 结构性变化 | 2018 年的转债市场和 2026 年差异很大（个券从 200+ 到 500+），历史类比未必成立 | 这是样本外泛化问题，不是过拟合问题；应对方式是保持调整幅度小 |

### 9.3 调参系统结构

```
调参系统
├── baseline_run()              # 固定参数跑一遍
├── online_learning_run()       # 在线学习跑一遍
├── compare(baseline, online)   # 对比两条净值曲线
└── utils
    ├── similar_env_retriever   # 类似环境检索
    └── factor_ic_calculator    # 因子IC计算
```

不需要 TunerBase 抽象类，不需要 param_grid，不需要 anti_overfit 模块。

### 9.4 模式 A：固定参数回测（Baseline）

最简单的模式——不调参，跑一遍看效果：

```
result = backtester.run(default_config, start, end)
metrics = compute_metrics(result)
print(f"夏普: {metrics['sharpe']:.2f}, 最大回撤: {metrics['max_dd']:.1%}")
```

用途：建立基线，验证因子本身是否有 alpha。**如果固定参数跑不出合理结果（夏普 < 0.5），在线学习也救不了。**

### 9.5 模式 B：在线学习调权（Online Weight Tuning）

这是调参系统的唯一优化模式。在线学习属于调参系统而非策略内部，因为它本质上是：

> 用历史表现来调整参数（因子权重），然后把新参数给回测引擎跑，看效果好不好。

策略仍然是纯函数——每次拿到的 config 里 factor_weights 不同而已。

#### 9.5.1 核心流程

```
for each rebalance_point in timeline:

    (1) 截取 rebalance_point 之前的环境历史
    (2) 用当前环境向量检索类似历史环境（top-K）
    (3) 在类似环境样本上统计各因子 rank_IC
    (4) 计算 confidence → delta → weight_adjustment
    (5) 生成新的 factor_weights
    (6) 构造新的 config（只改 factor_weights，其他不变）
    (7) 调用回测引擎跑 [rebalance_point, next_rebalance_point] 这一段
    (8) 拼接结果

最终得到一条"动态调权"的净值曲线
```

#### 9.5.2 类似环境检索

输入：当前环境向量 `env_t`，历史环境序列 `env_history[0:t]`

```
distance(env_t, env_i) = sqrt(
    w_eq * (eq_t - eq_i)^2 +
    w_bd * (bd_t - bd_i)^2 +
    w_tr * (tr_t - tr_i)^2
)
```

距离权重：`w_eq=0.3, w_bd=0.3, w_tr=0.4`

- 取距离最近的 top-K（K=50，比原设计增大以获得更稳定的 IC 估计）
- 样本间至少间隔 5 个交易日（避免同一段行情重复取样）
- 类似样本 < 30 个 → 跳过在线学习，使用基准权重

#### 9.5.3 因子有效性统计

对每个类似环境时点 i：
- 取该日因子分数截面
- 取该日之后 holding_period（40 日，比原设计增大以减少噪音）的各标的收益
- 计算每个因子与后续收益的 Spearman rank_IC

汇总：
```
mean_IC = mean(所有样本的 rank_IC)
std_IC = std(所有样本的 rank_IC)
IC_IR = mean_IC / max(std_IC, 0.01)
```

#### 9.5.4 Confidence 与 Delta

```
confidence = sigmoid(IC_IR * scale_param)        # scale_param 默认 3.0
baseline_confidence = 全历史（不区分环境）算出的 confidence
delta = confidence - baseline_confidence
```

#### 9.5.5 权重修正

```
online_adjustment = clip(delta * learning_rate, -max_adj, +max_adj)
# learning_rate = 0.10（更保守），max_adj = 0.03（单因子单次最多偏移3%）

new_factor_weights = normalize(clip(
    base_weights + env_adjustment + online_adjustment,
    0.05, 0.40
))
```

**参数收紧对照**（相对原设计）：

| 参数 | 原设计 | 收紧后 | 理由 |
|------|--------|--------|------|
| similar_env_k | 30 | 50 | 样本多一点，IC 估计更稳 |
| min_similar_samples | 20 | 30 | 不够就不调，回退固定权重 |
| learning_rate | 0.15 | 0.10 | 更保守 |
| max_adjustment | 0.05 | 0.03 | 单因子单次最多偏移 3% |
| holding_period | 20 | 40 | 用更长持有期算 IC，减少噪音 |

#### 9.5.6 回退机制

| 场景 | 处理 |
|------|------|
| 类似样本 < 30 个 | 跳过在线调权，只用环境偏移 |
| 某因子 IC 样本不足 | 该因子 confidence 回退到 baseline |
| 所有 delta 绝对值 < 0.01 | 不调整 |
| 训练期前12个月 | 纯基准权重 |

### 9.6 对比验证

```
def compare(baseline_result, online_result):
    """
    对比固定参数和在线学习的净值曲线。
    判断标准简单粗暴：直看夏普和最大回撤。
    """
    baseline_sharpe = sharpe(baseline_result)
    online_sharpe = sharpe(online_result)
    baseline_dd = max_drawdown(baseline_result)
    online_dd = max_drawdown(online_result)

    improvement = (online_sharpe - baseline_sharpe) / baseline_sharpe

    # 硬规则：在线学习必须比固定参数好至少5%
    if improvement < 0.05:
        推荐关闭在线学习
    else:
        推荐开启在线学习

    输出对比报告（含分年度明细），供人判断
```

**这不是自动化决策，是辅助人判断。** 跑一次固定参数、跑一次在线学习，对比净值曲线，人眼看看。

### 9.7 与债性保护的配合

可转债有债底，这是这个策略不需要在熊市清仓的根本原因。在线学习在这个背景下的角色是：

- 当环境偏债性时（bond_strength 高），在线学习可能把 Carry 和 Value 因子权重小幅上调
- 当环境偏股性时（equity_strength 高），在线学习可能把 Trend 因子权重小幅上调
- 无论如何，组合始终满仓或接近满仓——仓位不变，只变风格

**在线学习不是减仓信号，而是风格偏移信号。** 这与策略信条"环境调权重，不切策略"和"债性保护是天然的安全垫"完全一致。

## 阶段 2：主策略（纯函数层）

### 2.1 策略主入口

**文件**：`strategy/strategy_fn.py`

**接口**：

```
def strategy(snapshot: dict, config: StrategyConfig) -> Dict[str, float]:
    """
    主策略纯函数。
    输入：当日截面数据 + 参数
    输出：目标组合权重 Dict[cb_code → weight]
    
    不依赖任何外部状态。同样的输入永远返回同样的输出。
    """
```

**实现步骤**：

```
def strategy(snapshot, config):
    # Step 1: 环境
    env = compute_environment(snapshot, config)

    # Step 2: 因子
    factors = compute_factors(snapshot, config)

    # Step 3: 权重
    weights = compute_factor_weights(env, config)

    # Step 4: 打分
    scores = compute_scores(factors, weights)

    # Step 5: 组合
    raw_portfolio = build_portfolio(scores, snapshot, config)

    # Step 6: 风险约束
    final_portfolio = apply_risk_constraints(raw_portfolio, snapshot, config)

    return final_portfolio
```

**每个 Step 都是独立纯函数，定义在各自文件中。**

**测试要点（关键）**：
- **幂等性**：同一 snapshot + config 调用两次，输出完全相同
- **无副作用**：函数调用前后，没有任何全局/模块级变量被修改
- **无未来函数**：把 snapshot 中 trade_date 之后的数据替换为随机值，输出不变

```
    def run(self, config: StrategyConfig, start_date: date, end_date: date) -> BacktestResult:
        """用指定 config 运行完整回测（调参系统每次传入不同的 config）"""
```

**实现步骤**：

```
run(start_date, end_date):
    Step 1: 初始化
        - 拉取全量历史数据 → CleanedData
        - 初始化 SnapshotBuilder
        - 初始化 BacktestState（空仓，nav=1.0）
        - 生成交易日序列和再平衡日序列

    Step 2: 逐日循环
        for each trading_day in [start_date, end_date]:

            # 2a: 每日更新
            更新持仓市值
            计算当日组合收益
            state.update_nav(daily_return)
            state.update_risk_metrics(...)

            # 2b: 快系统检查
            extreme_action = extreme_protection.check(state, config)
            if extreme_action:
                执行强制减仓
                记录交易

            # 2c: 再平衡日逻辑
            if trading_day in rebalance_dates:

                # 构建 snapshot（注入 portfolio_hint）
                hint = state.get_portfolio_hint()
                snapshot = snapshot_builder.build(trading_day, hint)

                # 调用主策略
                target_weights = strategy(snapshot, config)

                # 权重平滑
                smoothed = smooth_weights(target_weights,
                                         state.prev_target_weights, config)
                state.prev_target_weights = target_weights

                # 调仓执行
                trades = execute_rebalance(smoothed, state, config)
                更新 state

            # 2d: 记录
            recorder.record(state.snapshot_for_record())

    Step 3: 返回 BacktestResult
```

**关键点**：
- 主策略调用处是唯一的 `strategy(snapshot, config)` 一行
- 回测引擎对策略是黑盒——即使换了一个完全不同的策略函数（只要签名相同），回测引擎不需要任何修改

### 3.3 调仓执行

**文件**：`backtest/execution.py`

**接口**：

```
def execute_rebalance(target_weights: Dict[str, float],
                      state: BacktestState,
                      config: StrategyConfig) -> List[dict]:
    """
    执行一次调仓，返回交易列表。
    内部处理：换手率控制、冷却期过滤、再平衡阈值、交易成本。

    """
```

**实现步骤**：

```
Step 1: 计算 delta = target - actual（逐标的）
Step 2: 冷却期过滤（卖出方向，非风险强制的跳过）
Step 3: 再平衡阈值（|delta| < 1% 的跳过）
Step 4: 换手率控制（turnover > max_turnover 则等比缩小）
Step 5: 计算交易成本（佣金 + 滑点，小规模券上浮）
Step 6: 生成交易列表，更新 state.holdings / actual_weights / cash
Step 7: 新买入标的设冷却期
```

### 4.3 在线学习调权

**文件**：`tuner/online_learning.py`

这是调参系统的唯一优化模式。它在每个再平衡点根据历史因子表现微调权重，本质上是"用过去训练，用未来验证"的天然 Walk-forward 结构。

```
def online_learning_run(engine: BacktestEngine, config: StrategyConfig,
                        start: date, end: date) -> BacktestResult:
    """
    在线学习调权：每个再平衡点检索类似环境，统计因子IC，微调权重。
    不继承基类，就是一个函数。
    """
    # 初始化
    retriever = SimilarEnvironmentRetriever(config)
    ic_calculator = FactorICCalculator(config)
    env_history = 预热阶段计算环境历史

    nav_segments = []

    for current_rb, next_rb in rebalance_pairs:

        # (1) 类似环境检索
        current_env = env_history.loc[current_rb]
        similar_dates = retriever.retrieve(current_env, env_history[:current_rb])

        # (2) 判断样本是否充足
        if len(similar_dates) >= config.get("tuner.min_similar_samples"):  # 30

            # (3) 因子有效性统计
            ic_result = ic_calculator.compute(similar_dates)
            baseline_ic = ic_calculator.compute_baseline(env_history[:current_rb])

            # (4) 计算调整量
            delta = {}
            for factor in ic_result:
                d = ic_result[factor].confidence - baseline_ic[factor].confidence
                delta[factor] = d

            # (5) 生成新权重
            new_weights = adjust_weights(
                base_weights=config.get("weight.base"),
                env_adjustment=compute_env_shift(current_env, config),
                online_adjustment=delta,
                learning_rate=config.get("tuner.learning_rate"),     # 0.10
                max_adj=config.get("tuner.max_adjustment"),          # 0.03
            )
        else:
            # 样本不足，回退到环境偏移（不做在线调权）
            new_weights = apply_env_shift_only(
                base_weights=config.get("weight.base"),
                env=current_env, config=config
            )

        # (6) 构造本段 config 并跑回测
        segment_config = config.override({"weight.base": new_weights})
        segment_result = engine.run(segment_config, current_rb, next_rb)
        nav_segments.append(segment_result.nav_series)

    # (7) 拼接所有段的净值
    full_result = concat_results(nav_segments)
    metrics = compute_metrics(full_result)
    print_summary(metrics)
    return full_result
```

**adjust_weights 内部逻辑**（对应上篇 9.5.5）：

```
def adjust_weights(base_weights, env_adjustment, online_adjustment,
                   learning_rate, max_adj):
    """
    三层叠加：基准 + 环境偏移 + 在线微调。
    在线微调被 learning_rate 和 max_adj 双重限制。
    """
    online_adj = {
        f: clip(delta * learning_rate, -max_adj, +max_adj)
        for f, delta in online_adjustment.items()
    }
    raw = base_weights + env_adjustment + online_adj
    return normalize(clip(raw, 0.05, 0.40))
```

**回退机制**（对应上篇 9.5.6）：

| 场景 | 处理 |
|------|------|
| 类似样本 < 30 个 | 跳过在线调权，只用环境偏移 |
| 某因子 IC 样本不足 | 该因子 confidence 回退到 baseline |
| 所有 delta 绝对值 < 0.01 | 不调整 |
| 训练期前 12 个月 | 纯基准权重 |

### 4.4 类似环境检索

**文件**：`tuner/similar_env_retriever.py`

```
class SimilarEnvironmentRetriever:
    def __init__(self, config: StrategyConfig)

    def retrieve(self, current_env: dict, env_history: DataFrame) -> List[date]:
        """
        返回 top-K 个最相似历史日期。
        K = config.get("tuner.similar_env_k")  # 50
        """
```

**实现步骤**：

```
Step 1: 过滤当前日期之前的记录（严格不看未来）
Step 2: 计算加权欧氏距离
        distance = sqrt(0.3*(eq_t-eq_i)² + 0.3*(bd_t-bd_i)² + 0.4*(tr_t-tr_i)²)
Step 3: 排序取 top-K（K=50）
Step 4: 间隔去重（<5个交易日的保留距离更近的）
Step 5: 去重后不足 min_similar_samples(30) → 返回空列表
```

**参数**（相对原设计收紧）：

| 参数 | 原设计 | 当前 | 理由 |
|------|--------|------|------|
| similar_env_k | 30 | 50 | 样本多一点，IC 估计更稳 |
| min_similar_samples | 20 | 30 | 不够就不调，回退固定权重 |

### 4.5 因子 IC 计算

**文件**：`tuner/factor_ic_calculator.py`

原文件名为 `factor_performance.py`（已改名以更准确反映用途）。

```
class FactorICCalculator:
    def __init__(self, config: StrategyConfig)

    def compute(self, similar_dates: List[date]) -> Dict[str, ICResult]:
        """
        对给定的类似环境时点，计算各因子的 rank_IC 统计量。
        holding_period = config.get("tuner.holding_period")  # 40日
        """

    def compute_baseline(self, full_history: DataFrame) -> Dict[str, ICResult]:
        """用全历史（不区分环境）计算 baseline IC，作为对比基准"""
```

**ICResult 结构**：

```
class ICResult:
    mean_ic: float       # rank_IC 均值
    std_ic: float        # rank_IC 标准差
    ic_ir: float         # mean_ic / max(std_ic, 0.01)
    confidence: float    # sigmoid(ic_ir * scale_param)，scale_param=3.0
```

**compute 内部流程**：

```
for each similar_date in similar_dates:
    (1) 取该日的因子分数截面（5个因子 × N个标的）
    (2) 取该日之后 holding_period(40日) 的各标的收益率
    (3) 对每个因子，计算 Spearman rank_IC

汇总：
    mean_IC = mean(所有样本的 rank_IC)
    std_IC  = std(所有样本的 rank_IC)
    IC_IR   = mean_IC / max(std_IC, 0.01)
    confidence = sigmoid(IC_IR * 3.0)
```

**参数**（相对原设计收紧）：

| 参数 | 原设计 | 当前 | 理由 |
|------|--------|------|------|
| holding_period | 20 | 40 | 用更长持有期算 IC，减少噪音 |
| learning_rate | 0.15 | 0.10 | 更保守，单步调整更小 |
| max_adjustment | 0.05 | 0.03 | 单因子单次最多偏移 3% |

### 4.6 对比报告

**文件**：`tuner/compare.py`

```
def compare(baseline_result: BacktestResult,
            online_result: BacktestResult) -> CompareReport:
    """
    对比固定参数和在线学习的两条净值曲线。
    不做自动化决策，只输出对比报告供人判断。
    """
    baseline_metrics = compute_metrics(baseline_result)
    online_metrics = compute_metrics(online_result)

    improvement = (
        (online_metrics['sharpe'] - baseline_metrics['sharpe'])
        / baseline_metrics['sharpe']
    )

    # 硬规则建议（仅参考，不自动执行）
    if improvement < 0.05:
        recommendation = "建议关闭在线学习（提升不足5%）"
    else:
        recommendation = "建议开启在线学习"

    # 输出对比报告
    print_comparison_table(baseline_metrics, online_metrics)
    print_yearly_breakdown(baseline_result, online_result)
    print(f"\n建议：{recommendation}")

    return CompareReport(
        baseline_metrics=baseline_metrics,
        online_metrics=online_metrics,
        improvement=improvement,
        recommendation=recommendation,
    )
```

**对比报告包含**：
- 全周期对比：夏普、最大回撤、年化收益、换手率
- 分年度明细：每年两种模式的夏普和回撤
- 净值曲线叠加图（可选输出为 PNG）
- 建议（在线学习夏普提升 < 5% 则建议关闭）

**这不是自动化决策，是辅助人判断。** 跑一次固定参数、跑一次在线学习，对比净值曲线，人眼看看。
