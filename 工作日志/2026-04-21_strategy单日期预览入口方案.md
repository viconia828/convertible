# 2026-04-21 strategy 单日期预览入口方案

## 1. 背景

截至 `2026-04-21`，`strategy` 第一阶段骨架已经落地：

- [strategy/service.py](C:/Users/ai/Desktop/可转债多因子/strategy/service.py)
- [strategy/engine.py](C:/Users/ai/Desktop/可转债多因子/strategy/engine.py)
- [strategy/portfolio.py](C:/Users/ai/Desktop/可转债多因子/strategy/portfolio.py)

当前代码层已经可以跑出“单日 snapshot -> 目标组合”，但还缺一个面向用户的轻量入口，方便直接查看：

- 当日环境分数
- 当日因子权重
- 当日目标组合

因此，本轮新增一个“strategy 单日期预览入口”。

## 2. 目标

新增一个轻量预览工具，用于单个交易日快速查看策略输出，而不进入回测层。

目标用户体验：

1. 用户输入一个交易日。
2. 工具按当前配置构造 snapshot 并运行 `strategy`。
3. 终端直接输出当日环境、因子权重、目标组合和提示信息。

## 3. 本轮范围

本轮只做：

- 一个 CLI 入口：
  - `tools/preview_strategy.py`
- 一个桌面批处理入口：
  - `策略预览.bat`
- 轻量终端展示，不导出 Excel
- README 用户入口补充
- 工具层测试

## 4. 本轮不做什么

本轮不纳入：

- 回测入口
- Excel 导出
- 图表或网页展示
- 调仓执行模拟
- 多日期批量策略预览
- `strategy` 新增复杂参数

## 5. 设计原则

### 5.1 只消费现有 strategy 主链

预览入口不重复实现策略逻辑，直接复用：

- `StrategyService.build_snapshot(...)`
- `StrategyService.run_for_date(...)`

### 5.2 输出要偏“读结果”，不是“读 DataFrame 调试”

这个入口面向日常实跑，所以输出应该先展示：

1. 基本信息
2. 环境分数
3. 因子权重
4. 目标组合
5. 提示/告警

而不是直接打印全量诊断表。

### 5.3 默认只做单日期

本轮不支持日期区间，避免入口目标漂移到“迷你回测”。

## 6. 入口形式

### 6.1 CLI

建议新增：

参数建议：

### 6.2 BAT

风格对齐现有：

- [环境打分.bat](C:/Users/ai/Desktop/可转债多因子/环境打分.bat)
- [因子打分.bat](C:/Users/ai/Desktop/可转债多因子/因子打分.bat)

## 7. 预览输出结构

建议输出分成五段。

### 7.1 基本信息

建议包含：

- `trade_date`
- `历史窗口请求起点`
- `历史窗口实际起点`
- `eligible_count`
- `selected_count`
- `cash_weight`
- `data_quality_hints` 是否存在

### 7.2 环境分数

固定输出：

- `equity_strength`
- `bond_strength`
- `trend_strength`

### 7.3 因子权重

固定输出：

- `value`
- `carry`
- `structure`
- `trend`
- `stability`

### 7.4 目标组合

展示 [strategy/portfolio.py](C:/Users/ai/Desktop/可转债多因子/strategy/portfolio.py) 生成的最终组合表，至少包含：

- `rank`
- `cb_code`
- `target_weight`
- `total_score`

### 7.5 提示信息

展示：

- `snapshot.data_quality_hints`
- `decision.diagnostics.notes`
- `alignment_summary`

其中 `alignment_summary` 建议只做简表输出：

- `calendar`
- `kept`
- `dropped`

## 8. 和现有导出层的关系

预览入口与导出层的关系应保持清晰：

- 环境/因子导出：
  - 面向多日窗口 + Excel
- strategy 预览：
  - 面向单日决策 + 终端结果

因此本轮不建议让预览入口去拼导出层的 `run_summary` 格式。

但提示口径上应尽量靠近现有风格，例如：

- 数据质量提示
- history coverage 提示
- refresh 模式提示

## 9. 测试要求

建议新增：

- `tests/test_strategy_preview_tool.py`

至少覆盖：

1. 成功路径下能输出环境、权重、组合三个核心区块。
2. 当 `StrategyService` 抛错时，入口返回非零退出码。
3. 交互缺失参数时不会崩溃。
4. 输出包含 `history_start_requested / used` 和 `cash_weight`。

## 11. 结论

本轮最稳的做法是新增一个轻量 CLI 预览入口，而不是提前做 Excel 或回测接入。

这样能尽快把 `strategy` 主链变成“用户可以直接实跑和看结果”的能力，同时又不会把范围拉进回测层。
